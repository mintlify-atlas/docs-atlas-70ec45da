Killa is a **small and lightweight state management library** for vanilla JavaScript and React. It is inspired by [Zustand](https://github.com/pmndrs/zustand) and [SWR](https://swr.vercel.app), combining a Zustand-style store API with fine-grained selector subscriptions that prevent unnecessary re-renders.

```bash
npm install killa
```

## Why Killa?

Most state management solutions require significant setup: providers, reducers, action types, or complex configuration. Killa takes a different approach — you create a store from a plain object and subscribe to exactly the state you need. Nothing more.

The internal state is **immutable by design**. Every call to `getState()` returns a deep clone, so there is no risk of accidentally mutating shared state across different parts of your application.

## How it compares

Killa occupies a deliberate middle ground between two popular libraries:

| | Zustand | SWR | Killa |
|---|---|---|---|
| **Primary focus** | Client state | Server/async state | Client state |
| **React required** | Optional | Yes | Optional |
| **Selector re-renders** | Yes | Yes | Yes |
| **Immutable reads** | No | N/A | Yes |
| **Internal actions** | Yes | No | Yes |
| **Persist middleware** | Via plugin | N/A | Built-in |
| **CDN / UMD build** | No | No | Yes |

If you have used Zustand before, Killa will feel immediately familiar. The main differences are:

- State reads always return deep clones — you cannot accidentally mutate the store.
- The `subscribe` method supports a selector as a second argument directly on the store (not just in React hooks).
- The `resetState` method lets you restore the initial state or replace it with a new one at any time.
- A UMD build is provided for direct use in browsers via `<script>` tags, without a build step.

## Package structure

Killa ships several entry points so you only import what you use:

| Import path | What it provides |
|---|---|
| `killa` | `createStore` — the main entry point |
| `killa/core` | Core `createStore` and TypeScript types |
| `killa/react` | `useStore` React hook |
| `killa/persist` | `persist` and `normalizeStorage` middleware |
| `killa/deep-equals` | Internal deep equality utility |
| `killa/helpers` | Internal helper utilities |
| `killa/constants` | Internal constants |

<Note>
  `killa/react` requires React >= 18.0.0. The `killa` and `killa/core` packages have no React dependency and work in any JavaScript environment.
</Note>
