`createStore` initializes a store from either a plain state object or an initializer function. It returns a frozen `Store<T>` object (in production and development; unfrozen only in test environments).

## Signature

```typescript
function createStore<T extends State, U = InitializerFn<T> | T>(
  initializer?: U,
  options?: Options<T>
): Store<T>
```

### Type parameters

| Parameter | Constraint | Description |
| --- | --- | --- |
| `T` | `Record<string, any>` | The shape of the state object. |
| `U` | `InitializerFn<T> \| T` | Inferred from the `initializer` argument. |

## Parameters

<ParamField path="initializer" type="T | ((get, set) => T)">
  The initial state. Accepts two forms:

  - **Plain object** — the object becomes the initial state directly.
  - **Initializer function** `(get, set) => T` — called once at creation time. Receives `getState` and `setState` so the initial state can reference store methods (e.g., for actions).

  Defaults to an empty object `{}` when omitted.

  <Warning>
    Throws `Error: "Store must be an object."` if the resolved initial state is not a plain object.
  </Warning>
</ParamField>

<ParamField path="options" type="Options<T>">
  Optional configuration object. See [Options](#options) below.
</ParamField>

## Options

The `Options<T>` interface:

```typescript
interface Options<T> {
  compare?: (a: unknown, b: unknown) => boolean
  clone?: (state: T) => T
  use?: ((store: Store<T>) => void)[]
}
```

<ParamField path="options.compare" type="(a: unknown, b: unknown) => boolean">
  Custom equality function used to determine whether state has changed before notifying subscribers.

  Defaults to Killa's built-in `deepEquals` implementation. Provide a custom function for shallow comparison or specialized equality logic.

  ```typescript
  const store = createStore({ count: 0 }, {
    compare: (a, b) => a === b // shallow equality
  })
  ```
</ParamField>

<ParamField path="options.clone" type="(state: T) => T">
  Custom clone function used when returning state from `getState()`.

  Defaults to `just-clone` for deep cloning. Override when you need custom serialization or when working with non-plain objects.

  ```typescript
  import structuredClone from '@ungap/structured-clone'

  const store = createStore({ items: [] }, {
    clone: (state) => structuredClone(state)
  })
  ```
</ParamField>

<ParamField path="options.use" type="((store: Store<T>) => void)[]">
  An array of middleware functions applied to the store after creation. Each function receives the `Store<T>` instance and may mutate it (e.g., to attach extra methods).

  Middleware is applied in array order.

  ```typescript
  import { persist } from 'killa/persist'

  const store = createStore({ count: 0 }, {
    use: [persist({ name: 'my-store' })]
  })
  ```
</ParamField>

## Return value

Returns a `Store<T>` object. In all environments except `NODE_ENV=test` the returned object is frozen via `Object.freeze`, so its top-level properties cannot be reassigned directly.

See [Store methods](/api/store-methods) for the full `Store<T>` interface.

## Initializer forms

### Plain object

Pass a plain object and it becomes both the initial state and the reference used by `resetState()` and `getServerState()`.

```typescript
import { createStore } from 'killa'

interface CounterState {
  count: number
}

const counterStore = createStore<CounterState>({ count: 0 })

counterStore.getState() // { count: 0 }
```

### Initializer function

Use a function when your initial state needs to embed actions or close over the store's own `setState`.

```typescript
import { createStore } from 'killa'

interface CounterState {
  count: number
  increment: () => void
  decrement: () => void
}

const counterStore = createStore<CounterState>((get, set) => ({
  count: 0,
  increment: () => set(() => ({ count: get().count + 1 })),
  decrement: () => set(() => ({ count: get().count - 1 })),
}))
```

The `InitializerFn<T>` type:

```typescript
type InitializerFn<T> = (
  getState: Store<T>['getState'],
  setState: Store<T>['setState']
) => T
```

## TypeScript examples

### Typed store with options

```typescript
import { createStore, type Options } from 'killa'

interface UserState {
  id: string | null
  name: string
  email: string
}

const options: Options<UserState> = {
  compare: (a, b) => JSON.stringify(a) === JSON.stringify(b),
}

const userStore = createStore<UserState>(
  { id: null, name: '', email: '' },
  options
)
```

### With middleware

```typescript
import { createStore } from 'killa'
import { persist } from 'killa/persist'

interface PreferencesState {
  theme: 'light' | 'dark'
  language: string
}

const prefsStore = createStore<PreferencesState>(
  { theme: 'light', language: 'en' },
  {
    use: [
      persist<PreferencesState>({ name: 'user-preferences' })
    ]
  }
)
```

## Error handling

<Warning>
  `createStore` throws synchronously if the resolved initial state is not a plain object:

  ```typescript
  createStore(42)       // Error: Store must be an object.
  createStore('hello')  // Error: Store must be an object.
  createStore(null)     // Error: Store must be an object.
  ```

  This check runs after the initializer function is called, so initializer functions that return non-objects also trigger the error.
</Warning>
